# 存储后端

本章节概览 WANOS 当前支持接入的存储后端类型，包括 S3 公有云/自建对象存储、本地磁盘路径、WebDAV 服务以及 OneDrive 等，并说明在管理后台中为这些后端完成配置时需要关注的字段与常见使用场景。

## 通用字段（所有后端类型）

在「新增后端 / 编辑后端」弹窗中，所有类型都会出现这些通用字段：

- **名称**
  - 用于在列表和日志中区分不同后端，例如：`S3-北京-1`、`本地-机房A`。
  - 建议包含用途和位置信息，方便后续维护。

- **类型**
  - 从下拉框中选择：`S3`、`本地`、`WebDAV`、`OneDrive` 等。
  - 选定后，下方会出现对应类型的配置区域。

- **副本数**
  - 表示**该后端自身**预计能提供的冗余能力（例如内部多副本 / RAID）。
  - 主要用于容量与可靠性评估，跨后端的逻辑副本数会在「副本与存储策略」章节中说明。

- **总容量**
  - 使用类似 `10 GB / 1 TB` 的形式填写，用于展示和调度时的容量估算。
  - 留空或设置为「无限容量」表示不做硬限制（仍然建议结合实际物理容量进行预估）。

配置完通用字段后，根据选择的类型继续填写对应的配置区域。

## 后端状态与数据策略（概览）

在「后端列表」页面中，你会看到每个后端的状态与策略信息，这些会影响系统如何使用该后端：

- **状态（Status）**
  - 通常会显示类似「健康 / 异常」的状态，用于指示最近一次健康检查结果。
  - 当某个后端长期处于异常状态时，建议排查网络、凭证或服务可用性，并视情况调整数据策略。

- **数据策略（后端策略）**
  - 在管理界面中可以为后端设置不同的数据策略，例如：
    - 正常接收新数据；
    - 只读、不再写入新数据；
    - 进入“驱逐”状态，逐步迁出存量数据；
    - 标记为离线，暂时不进行读写。
  - 这些策略主要影响调度和迁移行为，详细说明会放在「副本与存储策略」章节。

- **副本数**
  - 在后端列表中可看到每个后端配置的副本数，便于从整体上评估可靠性。
