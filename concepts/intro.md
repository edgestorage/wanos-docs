# 简介

`WANOS`（WAN Object Storage）是一个分布式对象存储网关与编排系统，旨在通过统一的 S3 兼容接口，整合多云、边缘及本地异构存储资源。它支持灵活的策略编排与自动化任务调度，特别适用于多云混合存储与边缘/中心协同场景。

- **统一 S3 接口**：基于 `versitygw` 提供 S3 兼容访问，屏蔽底层存储差异（支持 Local、NAS、S3、WebDAV、OneDrive 等）。
- **多后端编排**：支持混合云架构，可根据策略在不同后端间进行数据分层、迁移与多副本管理（Replication）。
- **边缘-中心架构**：Center 负责元数据管理与全局调度，Edge (DataNode) 负责就近数据读写与缓存，支持跨地域部署。
- **闭环与自动化任务**：内置对象搬迁、垃圾回收、一致性校验与副本修复等轻量级后台任务，简化运维复杂度。

## 适用场景示例

WANOS 更适合用在下面这类场景中：

- 你已经有一个或多个存储系统（本地磁盘、NAS、云对象存储、WebDAV/OneDrive 等），希望通过统一的 S3 接口对外提供服务；
- 有多块「零散的小盘 / 小容量存储」，希望在业务侧看起来是一个统一的大对象存储空间；
- 希望在「本地 + 云」之间做冷热分层或多副本编排，例如：热数据在本地，冷数据在云端；
- 希望在不改动业务的前提下，做后端迁移、扩容或下线（通过策略和后台任务逐步搬迁数据）；
- 需要一个轻量的中心节点统一管理多节点/多机房中的存储后端与任务调度。

如果你只需要一个单一的对象存储服务，而不涉及多后端编排或边缘/中心协同，那么直接使用云厂商或现有对象存储产品可能更简单；WANOS 更偏向在这些系统之上做统一网关与策略编排。

## 从这里开始

如果你是第一次接触 WANOS，可以按下面的顺序快速了解和上手：

1. **了解整体设计与对比**：
   - 阅读「[设计与对比](/concepts/design-and-comparison)」，了解 WANOS 解决的问题、设计取舍以及与其他方案的大致对比。

2. **理解架构与能力边界**：
   - 阅读「[架构设计](/concepts/architecture)」，了解 Center / Edge 的角色分工、多级缓存与任务调度等核心架构；
   - 阅读「[能力与场景](/concepts/capabilities)」，确认哪些场景适合使用 WANOS，哪些则不建议使用。

3. **动手搭建与体验**：
   - 按照「[安装](/manual/install)」章节在本地或测试环境中快速启动一个最小可用集群；
   - 参考「[使用](/manual/usage)」章节，通过 S3 客户端或 SDK 实际读写对象，体验基本能力。

4. **接入真实存储后端与策略**：
   - 阅读「[存储后端](/reference/backends)」，按文档说明接入你的 S3、本地、WebDAV 或 OneDrive 等后端；
   - 阅读「[副本与存储策略](/reference/storage-policy)」，根据业务的重要程度和成本要求配置合适的副本规则与数据策略。
